How to deal with missing data?

There are several simplistic approaches:

  • drop the entire column with .drop(): let's do that for the 'Cabin' column
  • drop the samples with missing data with .dropna(): let's do that for the 'Embarked' column
  • fill the data with .fillna(): let's do that for the 'Age' column
In [14]:
# Drop the Cabin column
df = df.drop(columns=['Cabin'])

# Drop the two lines with missing Embarked
df = df.dropna(subset=["Embarked"])

# Fill the data without age with mean age
df['Age'] = df['Age'].fillna(df['Age'].mean())

III.1 Categorical data¶

Categorical data has to be one-hot-encoded: meaning every class has to be transformed into 0 and 1.

We can do that with the pandas function pd.get_dummies()

1 1 0
2 1 0
3 1 0

This transformed the strings 'male' and 'female' into 0 or 1.

But isn't there redundancy in the information here? We can remove one column automatically with the option drop_first=True:

In [19]:
pd.get_dummies(df['Sex'], drop_first=True).head()

1 0
2 0

So that now we have well encoded our gender attribute to a binary variable.

What about non-binary categorical data?

  • Q is now replaced by (0, 1, 0)

III.5 Splitting the data¶

It is particularly important to separate the data that you use for training your model, and the data that you use for testing your model.

It would be like a student practicing on sample questions for his/her exams, and being tested on the exact same question.

You need to be tested on new questions in order to make sure you learnt correctly.

We usually split it in a training set and a test set with the associated repartition: 80% - 20%.

This convention allows to retrieve enough data to learn from it, and a small test subset in order to understand the performance.

To recap, we have:

  • a Training set (X_train, y_train): A set of observations used for learning. It fits the parameters (or weights) of the model.
  • a Test set (X_test, y_test): A set of observations (unseen during training) used only to assess the performance of the model.
